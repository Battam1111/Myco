# vestigial_organ 机制设计决策

## 背景

Myco 的免疫系统（23 维 lint + hunger signals）能检测文档断链、数值漂移、格式违规、知识腐烂，但无法检测"代码本身腐烂了"——模块存在但没人 import，函数定义了但没人调用。upstream.py 在项目里藏了好几周，677 处引用，如果用户不提没有任何机制能发现。

## 解决方案

新增 `vestigial_organ` hunger signal：
- 扫描 src/myco/ 下所有 .py 模块
- 排除入口点（cli.py, mcp_server.py, __init__.py, __main__.py）
- 检查每个模块是否被 src/myco/ 或 tests/ 中的其他文件 import
- 零导入者 = 退化器官 → 触发 hunger signal
- Agent 审查后决定：保留（给出理由）或切除（删除）

## 局限

- 只检测模块级别的零导入，不检测函数级别的死代码
- MCP 包装器内的 lazy import 会让模块"看起来活着"（如 discover.py 被 mcp_server.py 导入但实际无人调用 MCP tool）
- 更深层的"imported but never invoked"需要运行时使用追踪，是下一步

## 仿生学

退化器官（vestigial organ）= 曾经有功能但进化中失去用途的结构。人类的阑尾、智齿。代码中的 upstream.py、discover.py。检测它们是免疫系统的职责。
