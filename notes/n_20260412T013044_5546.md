## Key engineering patterns confirmed in hermes source

1. **COMMAND_REGISTRY** — frozen dataclass in `hermes_cli/commands.py`, ~60 CommandDef entries with name/description/category/aliases/args_hint/subcommands/cli_only/gateway_only/gateway_config_gate. ALL consumers derive from this: CLI dispatcher, Telegram BotCommand menu, Slack subcommand map, gateway help, autocomplete. Adding an alias = one-line edit to tuple. Field gating handles surface-specific exposure.

2. **HERMES_HOME set before imports** — `main.py::_apply_profile_override()` runs at the VERY TOP before any hermes import, pre-parses sys.argv for `--profile`, sets `os.environ[HERMES_HOME]`, then imports proceed. Enforced by code structure + comment "Many modules cache HERMES_HOME at import time". 119+ call sites of `get_hermes_home()`. Scar: PR #3575 fixed 5 hardcoded-path bugs → birthed this policy.

3. **SQLite + FTS5 schema** — `hermes_state.py`, 3 tables (schema_version, sessions, messages) + FTS5 virtual table `messages_fts` with auto-sync triggers (INSERT/DELETE/UPDATE). SCHEMA_VERSION counter + ALTER TABLE migrations each wrapped in try/except (v2 added finish_reason, v5 billing/cache cols, v6 reasoning fields). Thread-safe via single lock + BEGIN IMMEDIATE.

4. **Multi-layer config loader** — `load_config()` in `hermes_cli/config.py:2108`: deep-copy DEFAULT_CONFIG → `_deep_merge` user YAML → `_normalize_*` legacy keys → `_expand_env_vars` (${VAR} → os.environ). Single pipeline, single function to trace.

5. **`hermes doctor`** — `hermes_cli/doctor.py`, ~20 categories of checks: Python env, required/optional pkgs, config files (existence+version+migration), OAuth providers, directory structure, SQLite state.db (sessions count + WAL size), gateway systemd linger, external tools (git/rg/docker/ssh/node), live API connectivity (OpenRouter/Anthropic/OpenAI), npm audit. `--fix` auto-repair. Each check has check_ok/check_warn/check_fail/check_info outcome.

6. **Atomic YAML/file writes** — `atomic_yaml_write` helper referenced from `hermes_cli/config.py:2251`. Pattern: write to `.tmp` file, atomic rename to target. Applied throughout: sessions.json (PR#611), cron jobs (PR#146), .env (PR#954), process checkpoints (PR#298), skill files (PR#551). v0.2 release notes literally titled "Atomic Writes — data loss prevention".

7. **Managed mode awareness** — `is_managed()` checks HERMES_MANAGED env var or `.managed` marker file. In managed mode (NixOS), `_secure_file`/`_secure_dir` skip chmod (pkg mgr owns perms). HERMES_HOME_MODE env var for override (0701 for nginx traversal).

8. **Centralized logging** — `hermes_logging.py`, idempotent `setup_logging()`, `RotatingFileHandler` (5MB × 3 backups), `RedactingFormatter` (scans log lines for API keys/tokens before disk write), agent.log (INFO+) + errors.log (WARNING+), profile-aware paths, third-party noise loggers silenced. Lazy RedactingFormatter import to avoid circular deps.

9. **Error classification pipeline** — `agent/error_classifier.py`, `ClassifiedError` dataclass (reason, status_code, retryable, should_compress, should_rotate_credential, should_fallback). Priority-ordered classification: provider-specific patterns → HTTP status → error codes → message patterns → transport heuristics → fallback. 14+ FailoverReason variants.

10. **Jittered retry seeded from time+counter** — `agent/retry_utils.py::jittered_backoff`. Thread-local monotonic counter XORed with time_ns, `Random(seed).uniform()` for jitter. Prevents thundering herd on coarse clocks.

11. **Three-phase context compression** — `agent/context_compressor.py`. Phase 1 cheap prune (old tool results, no LLM call). Phase 2 protect head + tail by token budget. Phase 3 structured LLM summary of middle (Goal/Progress/Decisions/Files/Next). Iterative re-compression injects previous summary.

12. **Skill file anatomy** — `skills/*/SKILL.md` with YAML frontmatter (name/description/version/author/license/platforms/metadata.hermes). Filesystem IS the registry. Loader scans `rglob(SKILL.md)`, parses frontmatter, gates by platforms. Platform-aware via `skill_matches_platform()`.

13. **Self-registering ToolRegistry** — `tools/registry.py`. `registry.register(name, toolset, schema, handler, check_fn, is_async, ...)` called at module import. `get_definitions()` filters by `check_fn()` at schema-definition time (lazy availability). 90+ tools, no manifest.

14. **PII redaction via prefix regex** — `agent/redact.py`, 40+ `_PREFIX_PATTERNS` (sk-/ghp_/AKIA/github_pat_/telegram bot<n>:<token>/RSA private keys/db connection strings). Short tokens (<18 chars) fully masked; longer preserve first6+last4 for debuggability. Applied at logging time, not input time.

15. **BasePlatformAdapter ABC** — `gateway/platforms/base.py:726-944`. 20+ platform subclasses (Telegram/Slack/Discord/WhatsApp/Signal/Feishu/DingTalk/WeChat/SMS/Email/Webhook/HA/BlueBubbles) share 600+ lines, override only ~150 platform-specific. Per-session `asyncio.Event` for interruption, retryable vs fatal error tracking, SSRF redirect guards, `safe_url_for_log`.

16. **AGENTS.md as policy scar document** — 469 lines enumerating Profile-Safe Code invariants, Prompt Caching Integrity rules, Working Directory Behavior, Background Process Notifications, Shell/Terminal DO-NOTs (simple_term_menu ghosting, `\033[K` leak under prompt_toolkit), Dangerous Pitfalls (`_last_resolved_tool_names` process-global, cross-tool schema refs break models). Every rule has a scar origin.

17. **Release notes v0.2-v0.8 as scar archive** — each release enumerates bugs + PR numbers. Death-spiral detection born from #4750/#2153 (compression infinite loop). Thinking-budget exhaustion detection (#3444). Event loop closed fix three-layer (#3398). Supply chain removal of compromised litellm (#2796, #2810, #2812, #2816). OpenAI model reliability (#3528) — GPT_TOOL_USE_GUIDANCE. Context-exceeded errors with guidance (#4155, #4061).

18. **Test isolation via autouse fixture** — `tests/conftest.py::_isolate_hermes_home` redirects HERMES_HOME to `tmp_path/hermes_test` with subdirs pre-created. Plugin singleton reset. Env vars cleared. Profile tests pattern: mock Path.home() AND set HERMES_HOME env var (both). SIGALRM 30s global test timeout (Unix only). Integration tests marked `@pytest.mark.integration`, excluded by default via `addopts="-m 'not integration' -n auto"`. 3000+ tests run with pytest-xdist parallelism.

19. **Pinned deps with uv.lock hashes** — `pyproject.toml` pins all ranges. `[project.optional-dependencies]` groups: modal, daytona, dev, messaging, cron, voice, pty (with `sys_platform` markers), termux (Android subset), all. Supply chain audit workflow scans PRs. Entry points: hermes, hermes-agent, hermes-acp.

20. **Learning loop — interval-driven not signal-driven** — `run_agent.py:7486` `_turns_since_memory` counter, every N turns spawns background review with `_MEMORY_REVIEW_PROMPT` (semantic, LLM decides what to save). Skill review on tool iteration threshold. Background review agent forks context + shares memory store, runs in thread, doesn't block user latency.

## Myco gap audit (direct source probe, 2026-04-12)

1. **tests/** — 0 test files across 9100 lines of production code. Only test files in tree are inside `forage/repos/hermes-agent/tests` (Wave 23 dogfood discovered `pytest -x` crawls into it and fails on `ModuleNotFoundError: acp` → Wave 23 scoped to `<root>/tests/`). NH: zero test infrastructure.

2. **pyproject.toml** — 1 runtime dep (pyyaml), 1 optional extra (mcp). NO dev deps. NO `[tool.pytest.ini_options]`. NO `[tool.ruff]`. NO `[tool.mypy]`. NO `[tool.coverage]`. Single entry point (myco). NO platform markers. NO lock file (no uv.lock / poetry.lock).

3. **Atomic writes** — 1 site in entire codebase (`forage.py:183` `tmp_path = path.with_suffix(.yaml.tmp)`). `notes.py` and `upstream.py` write direct. Concurrent `myco eat` CAN corrupt the index.

4. **File locking** — 0 sites. No fcntl / filelock / portalocker. Concurrent `myco eat` + `myco digest` CAN race.

5. **Error taxonomy** — 2 Exception subclasses total: `MycoProjectNotFound` (Wave 20, notes.py:62), `UpstreamError` (upstream.py:72). No taxonomy of UserInputError / InvariantViolationError / SchemaError / IOError / BugError. No structured `.as_json()` error envelope.

6. **CLI duplication** — `cli.py` has 14 `if args.command == X` dispatch branches, ~500 lines argparse boilerplate. `mcp_server.py` has 9 `@mcp.tool` handlers. SAME 9 verbs (eat/digest/view/hunger/lint/status/log/reflect/retrospect) defined TWICE with different call signatures. Change one → must manually change the other. Confirmed: no single-source command registry. This is exactly the pre-factor-when-MCP-lands flag from the Wave 9 digest.

7. **Release notes** — NONE. Only `docs/contract_changelog.md` at contract level. Per-wave discipline lives in `log.md` milestone entries (readable but not versioned docs).

8. **Health check / doctor** — 0 sites. No `myco doctor` / `myco health` / `def check_*` function. Wave 20/21/24 doctrine "silent sensors are worse than crash" + "two sensors must agree" is the birth-pain of the missing doctor surface.

9. **Logging** — 0 uses of Python `logging` module. 255 `print()` calls across `src/myco`. No log file, no rotation, no redaction. User-facing output and diagnostic output conflated in same stdout stream.

10. **Config loading** — `_find_project_root()` walk-up pattern exists but is per-command, not env-var scoped. `MYCO_ALLOW_NO_PROJECT=1` escape hatch exists (Wave 20). No deep-merge of kernel defaults + instance overrides; `_canon.yaml` is read as-is.

## The substrate-vs-runtime boundary (what NOT to absorb)

Hermes is a runtime. Myco is a substrate. These ARE NOT portable to Myco:
- agent loop / tool dispatch / conversation state machine
- gateway platforms (Telegram/Slack/etc.)
- cron/scheduler for recurring execution
- `~/.hermes/` home-scoped state (Myco is project-scoped by design)
- OAuth/credential pool
- Context compression at runtime (Myco does compression at substrate level via notes lifecycle)
- 6 terminal backends
- MCP clients (we already have the forage channel for upstream intake)

What IS portable — engineering craftsmanship transferable to a markdown+YAML+lint substrate:
- Testing discipline (tests/, conftest autouse, markers, SIGALRM timeout, xdist parallel)
- Atomic writes (temp+rename for ALL state mutations)
- Error taxonomy with classified recovery hints
- Structured logging with rotation + secret redaction
- Command registry (single source → CLI + MCP + future gateway + future completion)
- Health check command (doctor)
- Release notes (scar archive, per-wave)
- AGENTS.md dangerous pitfalls + scar inventory extension
- pyproject.toml [tool.*] discipline (pytest/ruff/mypy/coverage/optional-dependencies)
- Atomic schema migrations with version counter
- Lazy availability via check_fn pattern (for lint rules, forage sources, future plugins)
