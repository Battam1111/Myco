## 9 decisions that become canon (for Wave 28 to implement)

**D1 — Unit of compression**: **tag-scoped** (primary) via `myco compress
--tag <TAG>` + **manual bundle** (escape hatch) via `myco compress <id1>
<id2> <id3>`. Rejected: semantic cluster (parametric, violates anchor #2),
time-window (too coarse), status-scoped (foldable into tag + `--status`).

**D2 — Trigger**: **manual invocation** (always allowed) + **friction-driven
hunger signal** `compression_ripe` (fires when ≥5 raw notes share a tag AND
oldest ≥7 days; thresholds in `_canon.yaml::system.notes_schema.compression`).
Rejected: scheduled cron (violates anchor #6 selection loop; substrate is
not a daemon host per `docs/open_problems.md §7`).

**D3 — Output shape**: **consumptive with audit**. One new `extracted` note
linking back to N originals via `compressed_from`; originals become
`excreted` with `compressed_into` back-reference. Rejected: additive
(doesn't reduce attention), wiki synthesis (conflates surfaces),
in-place promotion (loses synthesis).

**D4 — Audit trail**: mandatory frontmatter on output: `compressed_from`
(list), `compression_method` (manual/hunger-signal/auto), `compression_rationale`
(required prose — agent cannot `compress` without explaining), `compression_confidence`
(self-reported 0.0-1.0). Mandatory on inputs: `compressed_into`, optional
`pre_compression_status`. **New L18 lint dimension** (Wave 28) enforces
bidirectional link integrity. Cascade rejected at schema time: any note with
`compressed_from` field cannot itself be a compression input.

**D5 — Reversibility**: **data preserved, verb deferred**. Inputs remain on
disk with `excreted` status + back-references. `myco uncompress` is Wave 29+
territory. Optional `pre_compression_status` frontmatter records prior state
for future restoration. Language softened per Attack H — "preserved for
reversibility" not "reversible by design".

**D6 — Pipeline integration**: `myco compress` **folds anchor #3 steps
2+3+5+7** into one verb. Evaluate preserved via mandatory `--dry-run` flag
(Attack G partial land). Extract = compress output (extracted status).
Compress = the verb itself. Excrete = side-effect on inputs. Step 4
(integrate) remains separate as `myco digest --to integrated`. Post-Wave-28
anchor #3 impl_coverage estimated 0.43 → ~0.75.

**D7 — Non-functional contract**: **idempotent** (empty cohort = no-op),
**atomic** (all-or-nothing multi-file write via temp+rename), **concurrent-
safe** (snapshot cohort before writes; no file locking required MVP),
**cost bound** O(N) file ops where N = input count. Agent-adaptive policy
(32K vs 200K from anchor #4) is NOT implemented in Wave 28 — deferred to
possible open problem.

**D8 — Wave 28 scope recommendation**: bundle `myco compress` MVP + minimum
`src/myco/io_utils.py::atomic_write_yaml` helper (~30 lines) as ONE subsystem
("compress operation + its supporting atomicity"). Do NOT generalize atomic
writes across `notes.py`/`upstream.py`/`forage.py` in Wave 28 — those are
Wave 29+ as friction demands. Recommendation non-binding; Wave 28 craft's
Round 2 will re-check.

**D9 — Hallucination risk is bounded, not eliminated**: Attack F defense
produces 5 mitigations (data preserved for audit; bidirectional link
integrity; small-cohort friction floor via `ripe_threshold=5`; cascade
blocked at schema; single-point-audit acknowledged as structural weakness).
The residual risk (single human can't audit everything at scale) is
honestly registered as L1 limitation. Long-term mitigation requires a
second selection pressure source (multi-agent review, external source
verification, or statistical audit) — none in Wave 28 scope.
