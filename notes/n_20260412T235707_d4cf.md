# Mycelium as first-class architectural mechanism

Design direction: mycelium interconnection must be a FIRST-CLASS architectural mechanism, not a one-time manual cleanup.

Concrete implications:
1. L4 orphan detection should report by file category (docs/skills/wiki/notes), not one aggregate number
2. graph_orphans hunger signal should include category breakdown with specific action items per category
3. On every write (eat/digest/compress/inlet), the system could suggest related files to link — like "this note shares tags with X, Y, Z — consider adding cross-references"
4. The graph density metric (edges/nodes ratio) should be a tracked indicator in MYCO.md
5. New lint dimension candidate: "L23 Mycelium Density" — flag files with zero outbound links (not just zero inbound)

This is the "上升到精化的高层" principle: don't do it manually, build the mechanism that does it automatically and permanently.
