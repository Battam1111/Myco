## Why the pivot from atomic rename to additive phase 1

Wave 28 design craft §4.1 D2 proposed 11 atomic renames in one Wave 29 kernel_contract wave. Wave 29 execution craft §1 ran Phase A inventory and found:

- `_canon.yaml` alone appears 609 times across 100+ files (45-47 current-surface, ~55+ historical)
- Full atomic rename would require ~60-90 file edits bundled in one wave
- Single-response tool-call budget under marathon discipline cannot safely complete that without high risk of mid-wave context exhaustion leaving Myco in a physically broken state (imports pointing to nonexistent modules, canon keys missing from files that expect them, etc.)

The Wave 29 execution craft §4.3 L6 explicitly contemplated this pragmatic revision: *"If mid-execution context pressure forces a stop, the wave may need to split."* Phase A's real numbers made that contemplation concrete before execution started.

**Alternative considered: full atomic rename attempted**. Rejected because the failure mode (partial rename landing in a committed state) is worse than the failure mode of the additive approach (some names have both old and new entries, which is aesthetically inconsistent but functionally correct).

**Alternative considered: write new files, delete old files, update imports everywhere in sequence**. Rejected because the intermediate states between "write new file" and "delete old + update imports" leave the repo in a state where `myco lint` fails (because lint.py tries to import from `myco.notes` which both exists and is shadowed by `myco.metabolism`).

**Additive approach chosen**: create new modules as re-exports, add CLI verb aliases, commit, verify. The repo stays 100% functional at every step because the old names continue to work and the new names are pure additions. Physical file renames happen in later waves once the new import paths have matured.

## Decisions

**D1** — Wave 29 phase 1 ships: 2 new Python modules (metabolism.py + immune.py) + 2 new CLI verbs (molt + immune). Everything else defers.

**D2** — Old CLI verbs (`myco correct`, `myco lint`) are NOT deprecated. They remain primary, equal peers to the new verbs. Wave 28 D7's "transition alias with stderr warning" approach is SUPERSEDED by this wave's "equal-peer aliases, no deprecation warnings" approach. Rationale: marathon pivot to additive — no urgency to deprecate when both names work and the old names have universal-discoverability value for new users.

**D3** — NO contract bump. The kernel's public API gained 2 new modules and 2 new CLI verbs but broke nothing. This is a minor non-breaking addition that does not require `contract_version` bump. Contract version stays at v0.25.0.

**D4** — Wave 28 craft §4.1 D2 list of 11 renames remains as the DESIGN TARGET. Wave 29 phase 1 executes items 3+4+5 (module re-exports) and items 6+7 (CLI verb additions) additively. Items 1+2+8+9+10+11 (wiki/→hyphae/, _canon.yaml→_genome.yaml, canon key renames, lint labels) are deferred.

**D5** — Wave 29b is the next biomimetic work queued. Its scope: identify which deferred items are SAFE to do additively (canon key additions as new aliases, lint label extensions) vs which REQUIRE atomic commit (directory rename, file rename, old name removal). Wave 29b's craft will scope this decision.

**D6** — Marathon mode continues. Wave 30 can start using `myco.metabolism` and `myco.immune` import paths + `myco molt` + `myco immune` CLI verbs immediately, regardless of whether Wave 29b has shipped.

**D7** — Wave 28 craft §4.1 D5 (the requirement that "Wave 29's own craft re-checks the design under implementation pressure") was satisfied by Wave 29 execution craft §2 Attack N Defense — which was Phase A's real number discovery + pivot to additive.

## Future waves can now use biomimetic names

```python
from myco.metabolism import write_note, compute_hunger_report
```

And users/agents can run:

```bash
myco immune        # equivalent to myco lint
```

Both naming surfaces coexist. Future waves may rename the canonical implementation (moving code from notes.py/lint.py to metabolism.py/immune.py and making the old files thin re-exports in the other direction) or delete the old names entirely. Wave 29 phase 1 is the reversible additive seed.
