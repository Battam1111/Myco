# Hunger signals ordered by urgency

HungerReport signal ordering (most urgent first): raw_backlog > stale_raw > no_deep_digest > no_excretion > promote_ready > healthy. Rationale: constipation (too much raw) kills the substrate fastest — the agent stops being able to find anything. Starvation (no Gear 3) is slower but eventually fatal. Hoarding (no excretion) is the slowest failure mode but the one that correlates with long-term cognitive load the most.
