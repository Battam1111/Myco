User proposes a five-level top-down mapping to prevent drift: Story -> Visualized Story -> Formalized Vision -> Algorithmized Formalism -> Code/Implementation. The insight is that Myco currently has all these layers (anchors, vision.md, canon.yaml, lint dimensions, src/) but NO systematic traceability between them. When one layer changes, we manually hunt for downstream impacts. This is why 9 MCP stayed in READMEs for 10+ waves. We need a machine-readable mapping so changes at any level can trace their impact path. This is the most important structural improvement Myco can make for long-term anti-drift.
