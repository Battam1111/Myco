# CRITICAL: Myco is NOT a memory tool — it's a living knowledge organism

User critical insight: "Memory" as the selling point is WRONG. Memory is commoditized — Mem0, Zep, LangMem, MemPalace all do memory. It's table stakes, not a differentiator.

Myco's real differentiator is NOT memory. It's:
1. METABOLISM — knowledge has a lifecycle, gets digested, compressed, excreted. Nobody else does this.
2. IMMUNE SYSTEM — 23-dim self-verification that catches its own inconsistencies. Nobody else does this.
3. SELF-EVOLUTION — the engine itself mutates. Not just content evolving, but rules/lint/skills evolving. Nobody else does this.
4. ORGANISM — it's alive. It has hunger, it senses decay, it excretes waste. This is a fundamentally different CATEGORY.

The narrative should NOT be "your AI forgets → Myco remembers" (that's Mem0's pitch).
The narrative SHOULD be "your AI's knowledge rots, fragments, contradicts itself, and nobody checks → Myco metabolizes, verifies, evolves, and excretes."

The shift is not "memory vs no memory" — it's "dead knowledge vs living knowledge."
The enemy is not "forgetting" — it's "accumulation without metabolism" (tumors, not forgetting).
