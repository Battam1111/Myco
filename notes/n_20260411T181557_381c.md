# Multi-project mount ≠ multi-project focus

别把 ASCC mount 当成 Myco 的工作焦点：本次会话用户把 OPASCC 挂载进沙箱本意是给 Wave 9 upstream absorb 提供 bundle 来源和 dogfood 材料。但我 Wave 9 收尾之后开始 forage 工作时，差点又把 hermes-agent/mempalace 的 why 文本写成 'ASCC 的 Hard Contract 怎么怎么样'——实际上 Myco 本体开发才是现在的焦点，ASCC 只是一个下游使用者。触发场景：多项目挂载同时存在时，agent 容易把下游项目的 context 带到上游项目的设计决策里。期望行为：每次 'why/意图' 书写时，问一句'这个问题是 Myco 自己的问题，还是某个下游 instance 的问题？'—— 如果是后者，它不应该影响 Myco kernel 的 forage/digestion 优先级。workaround：用户明确 remind 了一次，我改。元教训：挂载点数量 ≠ 焦点数量；当前焦点的 SSoT 应该来自 MYCO.md 热区的 '当前阶段' 字段，不是文件系统里能看到什么。也可能需要在 session boot sequence 里加一步 '本次会话焦点 = ?' 的明确声明。
