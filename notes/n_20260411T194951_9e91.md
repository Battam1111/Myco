## What's actually interesting for Myco

**Terminology observation — "distill" as the verb.** This repo and the broader Agent Skills community use "distill" to mean "extract a behavioral pattern from a messy human source and compress it into a reusable skill." Myco uses "digest" to mean "move a raw note through a compression lifecycle toward integrated wisdom." **These are the same semantic action at different granularities:**

- `distill(person → skill)` = persona compression, output is a skill file + description
- `digest(raw note → integrated note)` = note compression, output is an integrated note with provenance chain

Myco already does "distill" at the project level via Gear 4 (Distill — extract universal patterns for future projects). The Gear 4 verb should probably be acknowledged as a synonym of the persona-distill community's "distill" verb in the README glossary, because agents trained on that community will trigger on "distill" faster than on "Gear 4".

**Category observation — no substrate-level distillation.** All 60+ entries in this list are per-individual persona skills. None of them distill a *substrate* or a *knowledge organization pattern*. This is validation that Myco's target (substrate-level distillation) is a different niche from Agent Skills' target (per-entity persona skills). **They do not compete.** They stack: a Myco substrate could contain and manage Agent-Skills-style persona skill files as an inbound channel.

**Framework observation — Agent Skills platform.** `agentskills.io` is the hub. This is a 6th signal on the skill-framework convergence pattern. Signal count for "LLM-callable skill packages as distribution unit" is now:

1. nuwa-skill (Claude Code Skill framework, MCP-adjacent)
2. pua (multi-platform skills: Claude Code / Cursor / Codex / VSCode / Kiro / ...)
3. gbrain (MCP server, not a skill framework per se)
4. hermes (runtime with tool registry, not skills)
5. mempalace (MCP server with 19 tools)
6. **agentskills.io ecosystem (this forage)** — an entire catalog of per-persona skills

Signals 1 + 2 + 6 are direct skill-framework signals. **3-signal convergence.** Myco should stop treating "skill" as an ambiguous term and decide: is Myco a skill provider (unlikely — it's a substrate), a skill container (plausible — forage could fetch skills from agentskills.io), or orthogonal (most accurate — Myco lints the project, skills act inside the project). The README positioning should explicitly say "Myco is not a skill framework. Skills run above Myco."
