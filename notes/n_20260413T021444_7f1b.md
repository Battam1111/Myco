# Decentralization + fork as selling point — synthesis with lazy-user persona

User insight on decentralization + fork incentive:

1. Since Myco is fully editable (git clone + pip install -e), every user gets a COMPLETE copy that can evolve independently. This is inherently decentralized — each Myco instance can diverge, specialize, and evolve in its own direction. Like biological speciation.

2. This is a unique selling point: maximum customization freedom. Not "configure via settings" freedom — "rewrite the kernel code" freedom. No other knowledge system offers this.

3. BUT user also notes: the target persona is "lazy people" (懒人). Lazy people don't fork. They want it to work out of the box. So the fork/customization angle is more of a DEVELOPER selling point, not the primary USER selling point.

4. The real synthesis: Myco works out of the box for lazy users (just talk), AND is fully forkable/customizable for power users who want to specialize it. Both audiences served by the same editable install model.

5. Decentralization angle: each project's Myco evolves differently based on that project's domain. Over time, a "forest" of specialized Myco instances emerges. Cross-project distillation (upstream protocol) lets them share nutrients — like a real mycorrhizal network across trees.
