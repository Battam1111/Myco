# Panorama #2 (post-Wave-19) — 9 new second-order holes NH-1..NH-9

第二次 panorama（post-Wave-17/18/19）产生 9 个 second-order 漏洞。核心发现：每层防御都依赖上一层的前置条件满足，grandfather 条款 + 静默 default 在 kernel/instance 版本漂移大时退化成静默绕过。

CRITICAL:
- NH-2 detect_contract_drift 在 ASCC（synced v0.8.0，kernel v0.18.0）返回 None 而非 REFLEX HIGH。根因：ASCC canon 没有 boot_reflex 块（Wave 13 之前的老 instance），boot_cfg.get('enabled', False) 默认 False → 函数第一行就 return None。grandfather 兼容退化成静默绕过。Wave 13 arc 对最需要的 instance 隐形。

HIGH:
- NH-1 myco hunger --project-dir . 在找不到 _canon.yaml 时静默用 raw root，报 total notes: 0 + healthy-ish signal，无警告。Agent 从错误目录跑会得到假确认。_project_root 应该 raise/exit 而不是 fallthrough。

MEDIUM:
- NH-3 boot brief 无 TTL 无 contract-change invalidation。L8 style lint 应该检查 brief mtime 晚于 canon mtime。
- NH-4 primordia 43/40 bloat 且每个 wave 单向写不读。craft 速度 > 压缩速度。需要 wave 结束时顺带 archive 一个 SUPERSEDED craft 的常规流程。
- NH-5 我在落地 Wave 18 的同 session 三次用 core.hooksPath=/dev/null 绕过刚装的 hook。hook 阻塞路径从未测试。习惯未被捕获。
- NH-6 本 session 至少两次自承错误（log type wave17→milestone；hook bypass 决策改变）0 次用 myco correct。Wave 19 人体工学门槛降到零但 volitional trigger 仍缺失（= craft §6 L-2 的实证）。
- NH-7 myco view 全战役零调用。agent 不是受众还是死表面，需要裁决。

LOW:
- NH-8 Wave 18 hook exit 1 分支未测，craft §5 dogfood 不完整。
- NH-9 hook 不跑 pytest；contract bump 落地 0 次 unit test。lint 是 schema 不是 correctness。

元观察：
- 两次 panorama 的洞不重复——第二次全部是第一次修复后才浮现的 second-order 问题，强烈支持 panorama 作为周期性 Phase 2 信号源（不是一次性审计）。建议 candidate 触发：每次 contract minor bump 后 / 每 10 waves / structural_bloat fire 时。
- NH-2 + NH-3 是同一个元模式：层叠 grandfather + 静默 default 退化成静默绕过。需要 'grandfather 漂移超过 N minor 后自动失效 + 要求 instance opt-in' 的 meta-规则。

触发场景：用户第二次要求 agent 做全景描述并从中找洞。
期望行为：反射弧的 second-order 依赖链有显式 lint（至少 L8-style brief freshness + grandfather drift ceiling）。
workaround：每次 contract bump 后手动跑 panorama。
