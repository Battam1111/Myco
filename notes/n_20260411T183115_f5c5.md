## What nuwa-skill actually is

A Claude Code skill that distills a named person (or topic) into a runnable persona-skill via a fixed 6-phase pipeline: **Phase 0 intake → 0.5 directory scaffold → 1 parallel 6-agent research swarm → 1.5 research review checkpoint → 2 synthesis → 2.5 synthesis review checkpoint → 3 skill construction → 4 quality validation → 5 dual-agent polish**. Output is a self-contained `.claude/skills/<person>-perspective/` directory that can be copied anywhere — sources, research notes, scripts, SKILL.md all in one tree. MIT licensed, active upstream, ancestor of Myco's own compiled-wisdom thinking (referenced in `docs/current/nuwa_caveman_integration_2026-04-07.md`).

## The three ideas Myco should borrow

**1. "Research dimensions derived from mental models, not fixed templates."** Nuwa's Phase 3 Agentic Protocol Step 2 is the highest-leverage design pattern in the whole skill. When a persona-skill receives a fact-requiring question, the *research dimensions it searches along* are auto-derived from that person's mental models, not a generic template. Munger → "look for moats, look for incentive structure, look for inverted risk". Feynman → "look for physical constraints, look for logical holes in official claims". **Myco's mapping**: `myco digest` currently just flips a status. A principled extraction protocol would make the extraction dimensions come from the note's own `tags` or `source.why`, so digesting a "friction-phase2" note uses friction-specific extraction axes (symptom, trigger context, expected behavior, workaround, meta-lesson) while digesting a "craft-conclusion" note uses craft-specific axes (rounds, confidence, decision-class, rebuttals). This is the kind of extraction shape that would make `--to extracted` mean something structurally.

**2. Mandatory review checkpoints with pause-and-show-summary.** Nuwa has four explicit human-in-loop pauses (0.5, 1.5, 2.5, 3 polish). Each pause shows a structured summary (a table in 1.5, a bullet list in 2.5) and waits for confirmation before proceeding. **Myco's mapping**: Myco's digestion pipeline is fully autonomous — `myco digest --to integrated` commits immediately without user review. For Gear 3 retrospectives and Gear 4 craft promotions, adding a "display-and-confirm" checkpoint (not a blocking one, but a summary-before-commit) would catch the kind of "I thought I was digesting X but I was really digesting Y" drift that the focus-drift friction note (`381c`) captured. This is a *Gear 2 candidate*, not yet actionable, but worth watching for a second independent signal.

**3. The "honest boundaries" requirement as a schema field.** Nuwa forces every generated skill to include a "what this CANNOT do" section: unknown-reaction-to-novel-problems, can't-replace-creativity, public-vs-real-gap, information-cutoff. Myco's craft_protocol captures confidence scores and rebuttals, but notes (including integrated ones) have no "failure mode" field — nothing that forces an author to state when the note's claim stops holding. **Myco's mapping**: low-priority candidate addition to `notes` schema — optional `boundary_conditions` frontmatter listing when this note's claim fails. Would make downstream reasoning about stale/misapplied notes sharper. Not urgent; log as a future schema consideration, not a v0.9.x item.

## What Myco has that nuwa doesn't

Four-channel metabolism (raw/digesting/extracted/integrated/excreted + forage inbound + upstream outbound), hunger signals, cross-instance upstream absorb, 15-dimension lint as a committed gate. Nuwa's pipeline is single-shot skill-generation with no long-lived state — each skill is a one-time artifact, no compounding across sessions. This is the biggest philosophical divergence: **nuwa distills humans into reusable thinking tools; Myco distills experience into reusable substrate**. Different target, different metabolism.

## What's explicitly NOT portable

The 6-parallel-agent research swarm. Myco currently has no agent-swarm primitive and adding one would be a Gear 4-scale architectural change, not a forage-driven lift. The nuwa approach optimizes for "distill a known person from public corpus in one pass"; Myco's approach optimizes for "accumulate and compound experience over many sessions". These are answering different questions. Don't retrofit nuwa's Phase 1 into Myco's digest flow — it would impose one-shot research semantics onto a metabolism that is fundamentally continuous.
