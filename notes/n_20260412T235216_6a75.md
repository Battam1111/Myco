# State of AI Agent Memory 2026 — Landscape Overview

## Key Findings (April 2026)

The AI agent memory space has matured significantly. Memory is now a first-class architectural component with its own benchmark suite (LOCOMO), dedicated research literature, and a rapidly expanding ecosystem.

## Benchmark Performance (LOCOMO)
- Full-context approach: 72.9% accuracy, 9.87s median latency (impractical)
- Mem0g (graph-enhanced): 68.4% accuracy, 2.59s p95 latency
- Mem0 (vector-only): 66.9% accuracy, 1.44s p95 latency
- RAG: 61.0% accuracy, 0.70s latency
- OpenAI Memory: 52.9% accuracy
- Key insight: selective memory achieves 91% lower p95 latency with modest accuracy sacrifice

## Three Memory Types Now Formalized
1. Episodic: what happened (conversation history)
2. Semantic: what is known (facts, preferences)
3. Procedural: how to do things (workflows, process patterns) — newly formalized

## Integration Landscape
- 13 agent framework integrations (LangChain, LangGraph, CrewAI, AutoGen, Google ADK, OpenAI Agents SDK, etc.)
- 3 voice integrations (ElevenLabs, LiveKit, Pipecat)
- 19 vector store backends
- OpenMemory MCP for local/privacy-first deployments

## Key Trends
- Voice agents driving fastest memory adoption (statelessness is painful in voice)
- Graph databases (Kuzu, Neo4j) displacing vector-only approaches for entity reasoning
- Async-by-default memory writes eliminating latency overhead
- Multi-scope memory model: memories scoped to user_id, agent_id, run_id, app_id
- Actor-aware memories in multi-agent systems

## Myco Positioning Implications
Myco occupies a fundamentally different niche from Mem0/Zep/LangMem. Those systems focus on conversation memory (remembering user preferences, past interactions). Myco focuses on knowledge metabolism (digesting, verifying, evolving project knowledge). The three-type taxonomy (episodic/semantic/procedural) maps loosely to Myco notes, wiki, and craft protocols — but Myco adds metabolism (digest lifecycle), immune system (lint), and self-evolution that none of these systems attempt.
