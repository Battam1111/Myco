# Forage article.md payload leaked past gitignore — payload-off-by-default fix

Forage payload-off-by-default 是 Wave 9 暴露的第三个空白：之前的 .gitignore 只排除了 papers/** / repos/** / articles/**/*.html，但 articles/**/*.md 是默认 tracked 的。这个不对称在本次 Wave 9 forage 真实落地前根本没人注意到，因为之前只有单元测试，从来没有 forage 过任何 article.md。直到 claude blog (proprietary) + karpathy gist (unknown license) 真的被放进 forage/articles/ 后才意识到：Myco public repo 不应该包含任何未消化的外部 raw 材料，更不应该包含 license 不明或 proprietary 的原文。根本原则：public Myco repo 代表的是 Myco 已经 processed + internalized 的产物（notes/、docs/、code），forage/ 只是 pre-digestion buffer，manifest trail（_index.yaml）进 git 作为审计线索，payload 永远 local-only，permissive license + 作者同意的情况下再 opt-in 用 git add -f 单条加入。修复：forage/.gitignore 改为 articles/** 全排除（对称于 papers/repos），forage/README.md rationale 三条补齐（undigested 非产物 / license 不均 / 压缩纪律 teeth）。触发场景：论坛/blog/gist 格式的 article.md，license 分布比 repo 更离散。期望行为：payload 永远 off-by-default，manifest always on。元教训：任何 substrate 的 public/private 边界都应该以'是否已消化'为轴，而不是以 mime-type 为轴。作为单独原则：public repo = 已内化的知识成品；private local = 未消化的原料。下次有新 substrate 上线时，默认选择应该是'新层永远不出本机'。
