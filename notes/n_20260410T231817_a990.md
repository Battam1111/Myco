# generate_id uses seconds + 4 hex, not uuid

Chose n_YYYYMMDDTHHMMSS_<4 hex> over UUID because: (a) human-scannable (you can tell when a note was eaten at a glance), (b) sortable by filename, (c) 4 hex chars = 65k collision space per second, more than enough for single-user or single-agent workloads. Cost: not globally unique across machines, but Myco notes are repo-scoped, so this is fine. If cross-repo merging ever becomes a thing, we can add a repo-id prefix without breaking existing ids.
