# Myco 全景使用描述 — Agent 第一人称触发流程 + 9 个结构性漏洞

## 核心发现
写这篇全景过程中发现 Myco 的所有反射弧设计依赖 agent 主动调用 `myco hunger` 或 `myco lint` 作为心跳。这两个命令是整个神经系统的感觉输入通道。一旦 agent 跳过它们——赶时间/忘了/context 被截断——所有下游反射弧失效。生物反射弧不需要意识决定运行感觉神经；Myco 的感觉输入是轮询式且 agent-initiated 的。这不是设计错误而是 CLI 工具宿主范式的物理限制。

## 9 个漏洞（按严重度）

### HIGH
- **H-1 (Boot 协议无强制)**: Wave 13 Boot Reflex Arc 只在 agent 主动跑 `myco hunger` 才触发。本次会话我从 Batch 3 续接，完全跳过 boot hunger——证据就是实际行为。反射弧声明了但感觉神经没接。
- **H-8 (Session end 非原子)**: agent 上下文边界模糊时 session end 不是原子时刻，Wave 14 nudge 依赖 boot 接收信号→闭环依赖 H-1。

### MEDIUM
- **H-3 (on-self-correction eat 未执行)**: Hard Contract 第 3 条特别条款规定自承错误必须同 turn 内 eat。本次会话至少 3 次自承错误（duplicate function stub、List import 缺失、m.end() vs m.group('body') bug），0 次 eat 为 friction-phase2/on-self-correction。无 lint 维度覆盖这条规则。
- **H-5 (L15 依赖手动 lint)**: Wave 12 craft reflex 依赖 agent 运行 lint 才能发红。直接 git commit 跳过 lint 时反射弧静默失败。无 git pre-commit hook 集成。
- **H-7 (Wave 14 LOW advisory 被 H-1 放大到接近 0)**: LOW severity 设计对（避免 W5 drive 泛化），但配套的'确保 agent 运行 hunger'上游约束从未兑现。
- **H-9 (upstream_scan_stale reader 不存在)**: Wave 16 写了 upstream_scan_last_run 时间戳写路径，但无 reader。已在 Wave 16 changelog L-1 声明 future work。Wave 13 A7 切片的后半段。

### LOW
- **H-2 (eat 触发纯意志力)**: 95% 时间碎想法丢失在输出里从未 eat。无 lint 相关性检查（如'会话出现 N 次"这很有意思"但 0 次 eat'）。
- **H-4 (L15 触发面白名单不全)**: 硬编码清单不含 docs/craft_protocol.md / docs/agent_protocol.md / docs/contract_changelog.md。Wave 15 我写 craft 纯粹是道德判断，不是 lint 强制。
- **H-6 (myco_log MCP vs shell 追加路径分岔)**: 沙箱 MCP 不总可用→shell HEREDOC 成为事实默认。功能满足 append-only，但丢失 MCP tool call 的可观测性。

## 建议 Wave 方向（非立即启动，仅供 Phase 2 设计参考）
- **Wave 17 修 H-1**: boot signal 机制——把 `myco hunger --for-boot` 输出嵌入 CLAUDE.md 热区渲染或 project instructions 模板，从'agent 必须主动调用'转成'agent 被动读到'
- **Wave 18 修 H-9**: upstream_scan_stale reader 作为 hunger 第四通道（完成 Wave 13 A7 另一半）
- **Wave 19 修 H-3**: `myco_log` MCP 在 agent append 明显 self-correction 类 meta 条目时自动建议 eat friction

## 元观察
写这份描述本身捕获了 3 个之前从未被任何 craft 捕捉到的漏洞（H-3, H-4, H-6）。这证明'让 agent 做全景第一人称描述'本身是一种 valuable Phase 2 信号源——可考虑作为周期性仪式（类似 Gear 2 反思但范围更大）。

触发场景: 用户要求 '告诉我你会如何使用 Myco 的全流程全景描述'
期望行为: 反射弧不再 100% 依赖 agent 主动心跳，至少关键信号（contract_drift, raw_backlog HIGH）能被动注入 agent 上下文
workaround: 无——需要 hosting 侧协作（project instructions 模板或 session hook）
