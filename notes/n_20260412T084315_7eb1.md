## D1 — SSoT principle (formalized from Wave 37 D1)

`len(FULL_CHECKS)` in `src/myco/lint.py` is the canonical
`LINT_DIMENSION_COUNT`. Every other surface that references the count is a
**downstream cache** that MUST be bumped in lockstep, and any cache that
isn't enforced by an automated check will silently rot. Wave 37 proved this
empirically: 15+ surfaces drifted from L0-L14 to L0-L18-claimed-as-L0-L14
across 7 waves (W30-W37) without any single moment where someone "saw"
the rot.

## D2 — Module-level FULL_CHECKS / QUICK_CHECKS tuples

Refactored `src/myco/lint.py::main()`'s local `checks = [...]` block into
module-level immutable tuples `FULL_CHECKS` and `QUICK_CHECKS`. Tuples
prevent accidental mutation; `main()` casts to list for its existing
local-mutable usage pattern. This enables L19 (and any future consumer) to
reference the SSoT without re-creating the dispatch table.

## D3 — L19 as a permanent structural lint

The fix shape MUST be a structural lint dimension, not a one-time bump
script. Wave 38 craft §2 Attack A defends this: a script run today
protects against today's drift but does nothing the next time a wave adds
L20. A structural check encoded as a lint dimension is the substrate-
native way to make any consistency invariant *permanently* hold — every
future `myco lint` re-runs it for free, and any wave that forgets to bump
downstream caches trips L19 immediately.

## D4 — HIGH/MEDIUM severity matrix

Surfaces are split into two severity classes:

- **HIGH** (4 surfaces): README.md, README_zh.md, README_ja.md, MYCO.md.
  These are user-facing first impression. Drift here is silent damage.
- **MEDIUM** (11 surfaces): CONTRIBUTING.md, wiki/README.md,
  src/myco/{cli,immune,mcp_server,migrate,init_cmd}.py,
  scripts/myco_{init,migrate}.py, docs/reusable_system_design.md,
  docs/adapters/README.md. These are maintainer-facing. Drift is
  internal friction but does not directly mislead users.

## D5 — Pinned-surface allowlist (NOT scanned)

L19 explicitly does NOT scan: log.md, docs/contract_changelog.md,
docs/primordia/*, notes/*, examples/ascc/*. These are append-only/pinned
per Hard Contract #2 — historical claims must be preserved verbatim
including their period-correct dimension counts. L19 also does NOT scan
src/myco/lint.py itself to avoid the own-file self-reference loophole.

## D6 — Five regex patterns

L19 uses exactly 5 regex patterns, each requiring a domain keyword to
minimize false positives:

1. `\bLint-(\d+)%2F(\d+)\b` — README badge URL
2. `\b(\d+)[- ]dimension(?:al)?\s+(?:lint|immune|consistency)` — English
3. `\b(\d+)\s*(?:维|次元)\s*(?:lint|L0|の|的)?` — CJK (covers 中文 + 日本語)
4. `\bL0[-–](?:L)?(\d+)\b` — L range (with quick-mode subset exemption)
5. `\b(\d+)/(\d+)\s+(?:green|绿|PASS|pass)` — pass ratio

Patterns 1, 2, 3, 5 expect integer == LINT_DIMENSION_COUNT. Pattern 4
expects integer == LINT_DIMENSION_COUNT - 1 OR len(QUICK_CHECKS) - 1
(quick mode L0-L3 is a legitimate subset reference).

## D8 — Four unit tests in tests/unit/test_lint_dimension_count.py

1. `test_l19_clean_substrate_passes` — base case (zero issues against
   a clean substrate that mirrors EXPECTED).
2. `test_l19_badge_drift_caught_high` — README badge drift surfaces
   as HIGH severity (not MEDIUM, not absent).
3. `test_l19_multi_pattern_co_match` — a single line matching multiple
   patterns produces ≥3 issues (no dedup collapsing).
4. `test_l19_severity_split_high_vs_medium` — identical drift in
   README (HIGH) vs CONTRIBUTING (MEDIUM) produces both severities.

Each test pulls EXPECTED from `len(FULL_CHECKS)` at import time so the
assertions also derive from the SSoT and never need hardcoded updates.

## D9 — mcp_server.py dispatch refactor (scope expansion)

`src/myco/mcp_server.py::myco_lint` was discovered to maintain its own
local `checks = [...]` list that had drifted to L0-L14 only and was
silently dropping L15-L18 from MCP-mode lint reports. This is exactly
the same SSoT-violation class L19 is designed to prevent at the
narrative layer, but at the dispatch layer it's a *behavioral* bug:
MCP users were running 15-dim lints while CLI users were running 19-dim.
Refactored to import `FULL_CHECKS, QUICK_CHECKS` directly from
`myco.lint` and dispatch through them. This makes mcp_server.py a true
downstream of the SSoT and eliminates the silent dispatch drift.

## Honest edges

- L19 only enforces the explicitly-enrolled 15 surfaces. New doc files
  that reference LINT_DIMENSION_COUNT must be manually added to
  `_L19_HIGH_SURFACES` or `_L19_MEDIUM_SURFACES` (Wave 38 craft §3 L2).
- Pattern 3 (CJK) is intentionally lenient on trailing context to catch
  both `19 维 lint` and `19 次元`. False positives possible if a doc
  uses `数字 维度` for unrelated geometry. Acceptable trade-off given
  zero such occurrences in current substrate.
- L19 cannot catch drift in surfaces that don't match any of the 5
  patterns (e.g. `nineteen-dimensional` spelled out in prose). Future
  wave can add patterns if such drift surfaces.
