Breakthrough insight from user: Myco should be packaged as a SKILL.md that auto-injects into Agent context at session start. PUA proves the pattern: skill files in .claude/skills/ are automatically loaded — Agent doesn't choose to use them, they're just THERE. Currently Myco relies on Agent choosing to read MYCO.md and call hunger — this is the same 'human must trigger' anti-pattern we identified in anchor #6 revision. A Myco SKILL.md would auto-inject: (1) compressed identity anchors, (2) boot ritual hunger(execute=true), (3) tool trigger conditions for all 18 tools, (4) traceability index reference, (5) anti-patterns. This closes open_problems section 7 (heartbeat dependency) at the root — Agent knows the protocol because it's injected, not because it remembered to read docs. This is the true Agent-First completion.
