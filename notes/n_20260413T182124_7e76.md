# 重复犯错模式清单 — 2026-04-13 session 反思

## 模式 1：sed 全局替换破坏 Python 代码
**错误**：用 `sed -i 's/myco import/myco assimilate/g'` 把 Python `from myco import __version__` 改成了 `from myco assimilate __version__`。
**根因**：sed 是文本替换，不理解 Python 语法。`myco import` 在 CLI 命令语境和 Python import 语境含义完全不同。
**机制**：重命名必须用语法感知的方式——先 `git mv` 文件，再用 Python AST 或精确的 `from myco.old_module` 模式替换 import，CLI 命令名用引号包裹的精确匹配 `"old_name"` → `"new_name"`。永远不要用裸 sed 替换 Python 关键字。
**防止**：未来实现 `myco rename` 命令，内置语法感知替换。

## 模式 2：--fix 自动修复破坏代码
**错误**：`myco immune --fix` 的 L12 修复逻辑从 .py 文件中剥离了 markdown 链接语法 `[title](path)` → `title`，但这些是 f-string 模板代码，不是文档内容。导致 `_auto_link_note` 功能被破坏、14 个测试失败。
**根因**：auto-fix 不区分"文档中的断链"和"代码中构造链接的模板"。
**机制**：auto-fix 永远不能修改 .py 文件（已实现 skip）。.py 中的 L12 问题只报告，由 Agent 判断是否修复。
**防止**：`_fix_l12_broken_links` 已添加 `.py` skip guard。

## 模式 3：扩展检测范围引入假阳性
**错误**：L12 扩展到 .py/.yaml 后，发现 .py 测试文件中的 mock 路径和 f-string 模板被误报为断链。
**根因**：测试 fixture 创建虚拟文件路径作为测试数据，这些路径本来就不存在。
**机制**：L12 跳过 tests/ 目录（已实现）。.py 文件只检查 backtick 路径，不检查 markdown 链接语法（已实现）。
**防止**：每次扩展检测范围前，先在当前项目上试跑，确认假阳性可控。

## 模式 4：修复后不验证关联影响
**错误**：仿生学重命名后声称"完成"，但 MCP annotations.title、adapter YAML、测试文件名、docs/ 引用全部遗漏。需要 10 路 Agent 审计才发现。
**根因**：rename 影响面远超 import 语句——还有字符串常量、文件名、文档引用、配置文件。
**机制**：每次重命名后必须运行完整 `myco immune`（含 .py/.yaml 扫描）+ 手动 grep 旧名确认零残留。
**防止**：三层免疫 + L12 扩展范围 + vestigial_organ。

## 模式 5：Agent 审计结论不交叉验证
**错误**：审计 Agent 说 Cowork 不支持 → 我直接删了 → 实际上 Cowork 是完全支持的。
**根因**：Agent 只看 detect_tools()，没看 _COWORK_SKILLS。我盲信了 Agent 的 BLOCKER 标签。
**机制**：每条 Agent FLAG/BLOCKER 必须用 grep/read 交叉验证。尤其是"删除"类建议。
**防止**：已写入记忆 + 吃进基质。

## 不可变规则

1. **sed 只用于文档**，Python 代码用精确的 import 模式替换
2. **auto-fix 永远不碰 .py**
3. **扩展检测范围前先试跑**
4. **每次重命名后 grep 旧名确认零残留**
5. **Agent 结论 = 建议，不是指令**
