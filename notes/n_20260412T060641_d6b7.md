# Wave 30 decisions: compress MVP D1-D8 (canon)

Wave 30 (kernel_contract, contract v0.26.0) — myco compress MVP. Decisions D1-D8 plus 7 known limitations are the binding canon for the verb's behavior, all derived from the compress_mvp_craft 3-round attack process.

D1 — Bundle the full subsystem in Wave 30. Ship compress_cmd.py + io_utils.py + L18 + compression_ripe + 5 tests + contract bump as ONE wave per Wave 27 D8. Splitting would create no-op intermediates per R1.1 defense: without L18 the torn-state surface escapes pre-commit; without compression_ripe anchor #4 agent-adaptive clause stays unimplemented; without atomic_write_text the multi-file write is less safe than notes.write_note.

D2 — Two-phase commit with output-first ordering + L18 backstop. Phase 1: build all content in memory (output text + per-input new texts). Phase 2: write output via atomic_write_text first (if it fails, zero torn state because no inputs touched), then iterate inputs. On any input write failure: collect, warn loudly to stderr with output id + recovery instructions. L18 catches any torn state on next lint run. Best-effort + lint backstop is the substrate's chosen integrity model — same as L1, L2, L8.

D3 — Cohort resolver default eligible-status = {raw, digesting} AND skip notes with compressed_from set. Dual filter is belt-and-suspenders defense against Wave 27 Attack F (audit-at-scale fabrication) and against the R2.3 silent-idempotence-violation that the first test run caught (output extracted note inheriting input tags re-resolved on second run, hit cascade rejection exit 3 instead of empty-cohort exit 4).

D4 — source: compress is a new enum value, not reused (resolves Wave 27 L3 in additive direction). Reasons: semantic distinction from promote (N->1 with structural data loss vs 1->1 rename), L18 lint surface efficiency (key off source==compress), discoverability for agents reading myco view --tag X. Added to valid_sources in both kernel and template canons.

D5 — compression_method stays free-form for Wave 30. L10 will not constrain values. Documented as L2 limitation. Reasons: optional field already, locking enum before friction-validating either string is premature, rationale field is the source of truth, future elevate-to-enum is trivial.

D6 — io_utils.py name is provisional pending Wave 29b biomimetic sweep. Wave 29b evaluates whether to rename to membrane.py (if all callers are metabolic) or other. Module docstring flags provisional status. Same additive-first-rename-later pattern Wave 29 used for metabolism.py.

D7 — L18 lint dimension count 18 -> 19, contract v0.25.0 -> v0.26.0. lint_compression_integrity registered in main() after L17. Lint count refreshed in module docstring. Contract bumped in kernel canon + template canon. ASCC syncs on next consumer pass.

D8 — compression_ripe is non-blocking advisory. Hunger signal returns plain compression_ripe text without [REFLEX HIGH] prefix. Per Wave 27 D2, friction-driven trigger not blocking. Anchor #6 (mutation+selection+transparency) requires agent retains selection authority — REFLEX HIGH would violate the selection loop.
