# Wave 40 — L21 Contract Version Inline Consistency lint (v0.31.0) — DECISIONS

**Wave**: 40

## Why this wave existed

Wave 37 D7 followup #3 (the third + final candidate from the Wave 37 manual sweep). Wave 38 closed #1 (L19 dimension count) and Wave 39 closed #2 (L20 translation mirror skeleton). The Wave 40 scout discovered the exact scar that justifies followup #3: `MYCO.md` lines 4, 22, and 159 plus `docs/adapters/README.md` line 14 were ALL still claiming `kernel contract v0.28.0` even though the canonical contract version had advanced through v0.29.0 (Wave 38) and v0.30.0 (Wave 39) without those inline narrative claims being touched. The Waves 38+39 manual sweeps caught dozens of dimension references and locale skeleton drifts but left these contract version inline claims behind because they live in different prose patterns ("kernel contract vX.Y.Z" rather than "L0-LXX" or "X-dimension"). L19 and L20 cannot see them — they look at counts and structures, not at version numbers embedded in narrative sentences. Without an automated check, every future wave that bumps the contract version risks repeating the exact silent rot Wave 40 was created to fix.

## Decisions D1–D9

- **D1 SSoT** — `_canon.yaml::system.contract_version` is the single source of truth. Every line matching the forward-looking pattern in an allowlisted file must mirror this value or be flagged.
- **D2 Detection algorithm** — Forward-looking regex `(?i)(kernel contract|当前 contract|current contract)\s*[:：]?\s*(v?\d+\.\d+\.\d+)` + 8-clause historical-marker filter (`_l21_is_historical`): operator skip-marker, date-in-parens, transition arrow, historical-event verbs, table row prefix. Bias toward false negatives over false positives. Code-fence aware.
- **D3 File allowlist** — HIGH = `MYCO.md`, `README.md`, `README_zh.md`, `README_ja.md` (entry-point + locale front-doors). MEDIUM = `docs/adapters/README.md`, `CONTRIBUTING.md`, `docs/architecture.md`, `docs/vision.md`, `docs/theory.md` (doctrinal/contrib). All other files (log.md, contract_changelog.md, primordia/, notes/, examples/ascc/, src/, tests/, scripts/) out of scope by design.
- **D4 Severity matrix** — HIGH for entry-point + locale tier, MEDIUM for doctrinal/contrib tier. Both fire under same lint pass; only explicit skip marker opts out.
- **D5 Skip-marker semantics** — `<!-- l21-skip -->` on the line immediately preceding the version claim excludes it. Same shape as Wave 39's `<!-- l20-skip -->` for parser consistency.
- **D6 Code-fence awareness** — Lines inside ` ``` ` blocks are skipped entirely. Code samples may show legitimate version literals that should not be enforced.
- **D7 Multi-language prefix** — Three accepted prefix forms: English `kernel contract`, English `current contract`, Chinese `当前 contract`. Other shapes (e.g. `Myco at vX.Y.Z`) are not caught — adding a new shape requires a craft if doctrinal.
- **D8 Test coverage** — 4 unit tests in `tests/unit/test_lint_contract_version_inline.py`: clean substrate, stale inline caught HIGH, historical marker skipped, skip marker respected. Suite count 30 → 34.
- **D9 Stale claim fix list** — The 4 currently-drifted claims that justified L21 are fixed in the same commit: MYCO.md:4, MYCO.md:22, MYCO.md:159, docs/adapters/README.md:14. All bumped from `v0.28.0` to `v0.31.0`.

## Closing doctrine

Wave 38 made the dimension count a single source of truth; Wave 39 made the locale README skeletons a single source of truth; Wave 40 makes the inline contract version a single source of truth. The Wave 37 D7 followup triple is now closed — three orthogonal silent-rot scar classes that the Wave 37 manual sweep ranked highest are all structurally impossible from this commit forward. Future waves can bump the contract version, add new locales, add new lint dimensions — and every narrative surface that mirrors any of those values will move in lockstep because the immune system now blocks single-surface drift at the lint boundary on three orthogonal axes.

## Limitations (honest)

- **L1**: forward-looking pattern recognizes only 3 prefix forms. Other legitimate shapes (e.g. `Myco at vX.Y.Z`, `contract version is now vX.Y.Z`) not caught.
- **L2**: historical-marker filter biased toward false negatives. A drifted line containing `landed` or a date stamp will be silently skipped — deliberate trade against operator-fatigue false positives.
- **L3**: package version (`__version__` / `pyproject.toml`) intentionally out of scope per Wave 8 re-baseline doctrine. Future `_L22_PACKAGE_VERSION` lint would need its own craft.
- **L4**: file allowlist hardcoded; new entry-point or doctrinal surfaces require a one-line patch + a contract bump.
