## 一、Myco 是什么

**一句话**：一个与 AI Agent 共生的自主认知基质，具备知识代谢和元进化能力。

**展开**：Myco 是 Agent 以外的一切。Agent（Claude、Codex、或任何未来的 LLM）提供推理和执行能力，Myco 提供持久记忆、结构化知识、过程经验和自进化机制。没有 Agent，Myco 是一堆死数据；没有 Myco，Agent 每次从零开始。两者结合形成一个跨会话、跨项目、跨平台的完整智能体。

**名字由来**：Myco 来自 mycelium（菌丝网络）——森林地下那张看不见的活网。菌丝不只是连接树木的管道：它分泌酶将落叶分解为养分，记住有效的生长路径并据此调整策略，根据需求将资源从丰裕区域调往匮乏区域，并与不同树种的根系都能形成共生。Agent 是地面上的树，Myco 是地下让整片森林成活的网络。

## 二、三条不可变宪法

这是 Myco 系统中唯一不可被进化替换的部分：

| # | 宪法 | 含义 | 为什么不可变 |
|---|------|------|------------|
| 1 | **入口可达** | 系统必须有一个任何 Agent 可定位、可自解释的入口 | 没有入口，Agent 无法启动系统 |
| 2 | **透明可理解** | 系统必须始终对人类保持可审查、可理解 | 失去透明性 = 人类失去选择压力 = 系统可能癌变 |
| 3 | **永恒进化** | 系统必须始终在进化路上，停滞即死亡 | 停止进化 = 失去核心特性 = 退化为静态知识库 |

**其他一切皆可进化**——知识如何组织、压缩策略、进化机制本身、人机协作方式、甚至系统使用的存储介质。

## 三、五大核心能力

### 能力 1：知识代谢

系统从外部世界自主发现、萃取、整合、压缩知识。不是被动接收，是**主动觅食**。

```
外部世界（GitHub / 论文 / 社区 / 工具）
  ↓ 自主发现（缺口触发 + 周期巡逻）
  ↓ 评估（相关性、质量、新颖性）
  ↓ 萃取（提取有用模式，不是复制原文）
  ↓ 整合（融入已有知识，不是简单追加）
  ↓ 压缩（保持精简，不臃肿）
  ↓ 验证（事实性、时效性）
系统 → 变得更强
```

不遗忘，但智能压缩。存储空间无限，但活跃时只加载相关知识。

**与 RAG 的区别**：RAG 是被动检索（有人提问才去找），知识代谢是主动觅食（系统自己决定去找什么、什么时候找）。

### 能力 2：元进化

不仅改进知识，还改进"获取/组织/使用知识的方式"本身。递归无上限。

**为什么这在我们的系统中特别可行**：进化对象是文件系统上的文本和结构——markdown 文件、Python 脚本、目录组织。这是 LLM Agent 最擅长操作的介质。相比 Meta Hyperagents（需要修改模型权重和训练循环），我们的元进化在技术上简单得多。

当前的手工规则（W1-W12、四层架构、Procedures 等）是实践中进化出来的合法 bootstrap。系统有权在后续进化中替换掉它们中的任何一个。

### 能力 3：自我模型

系统维护关于自身的模型——知道自己有什么、缺什么、什么在退化、什么没被使用。

| 层级 | 内容 | 自动化难度 |
|------|------|----------|
| A 库存清单 | 知识总量、分布、更新时间 | 低 |
| B 缺口感知 | 哪些领域缺乏知识（摩擦信号 = 缺口症状） | 中 |
| C 退化感知 | 哪些知识过时或结构不再适用 | 中 |
| D 效能评估 | 哪些知识被有效使用、哪些是"死知识" | 高 |

### 能力 4：跨会话连续性

让无记忆的 Agent 快速重建认知。层级存储模型：

- **热层**（活跃知识）：小、高度压缩、Agent 启动时直接加载
- **温层**（可检索知识）：中等规模、按需检索、结构化索引
- **冷层**（归档知识）：大规模、全文搜索、不主动加载

### 能力 5：Agent-Adaptive 通用性

内核不绑定特定 Agent / 项目 / 平台。运行时适配 Agent 能力（上下文窗口、工具能力、指令遵循度）。适配策略本身也可进化。

## 四、与已有系统的定位区分

| 维度 | Myco | Hyperagents (Meta) | Hermes Agent | Mem0 | 企业 KMS |
|------|------|-------------------|-------------|------|---------|
| **进化什么** | Agent 外部知识基质 | Agent 行为/策略 | Agent 技能 | 记忆存储 | 文档库 |
| **元进化** | ✅ 核心目标 | ✅ 部分 | ❌ | ❌ | ❌ |
| **知识代谢** | ✅ 核心目标 | ❌ | ❌ | ❌ | ❌ |
| **自我模型** | ✅ 目标 | 部分 | ❌ | ❌ | ❌ |
| **Agent-adaptive** | ✅ | ❌ | ❌ | ✅ | N/A |
| **质量保障** | ✅ | ❌ | ❌ | ❌ | 人工 |

**独特定位一句话**：现有系统要么进化 Agent 自身（Hyperagents），要么帮 Agent 记住过去（Hermes/Mem0）。**Myco 是第一个"能自我进化的外部认知基质"**。

## 五、人机协作模型

**系统做变异，人类做选择。**

系统不断产生变化——发现新知识、尝试新组织方式、试验新进化策略。人类不需要设计这些变化，只需要表达"这好 / 这不好"。选择压力来自人类，变异来自 Agent + Myco。

Agent 在这个模型中是"执行器"——它既执行项目任务，也执行 Myco 的自进化操作。Myco 是"基因组"——存储所有持久化的知识和策略，通过 Agent "表达"。

## 六、Bitter Lesson 立场

Rich Sutton 的 Bitter Lesson：利用算力的通用方法最终总打败利用人类知识的手工方法。

Myco 的回应：
- 当前的手工规则是**合法的热启动**——它们是实践中进化出来的，不是理论预设的
- 但它们不是神圣的——系统有权在进化中替换任何手工规则
- 唯一不可替换的是三条宪法
- 长期目标：系统自己发现的规则替代人类设计的规则

## 七、从 v0.x 到 v1.0 的跃迁

**v0.x（当前）**：
- 在 ASCC 项目中运行 8 天
- 人类驱动的元进化（人说"改进系统" → Agent 执行）
- 全 markdown 存储
- 单项目、单 Agent（Claude Cowork）
- 手工规则（W1-W12, P-001~P-007, 9 维 Lint）

**v1.0（目标）**：
- 系统自驱动的进化（自己发现改进点 → 自己执行 → 自己评估效果）
- 混合存储（md + 结构化数据，按效率选择）
- 内核与项目实例分离
- Agent-adaptive（不绑定 Claude）
- 知识代谢能力初步上线

**从 v0.x → v1.0 的核心跃迁**：从"人类驱动"到"系统自驱动"。这意味着需要实现自我模型的层级 A 和 B（库存清单 + 缺口感知），以及至少一种知识代谢触发机制（缺口触发）。
