# ROOT CAUSE: immune system only covers knowledge, not deployment

ROOT CAUSE: immune system blind spot

The 23-dimension lint only covers KNOWLEDGE consistency (markdown, yaml, notes, wiki, cross-references). It does NOT cover DEPLOYMENT/TOOLING consistency:
- init generates correct directories?
- scheduled task has all tools?
- settings.json has correct patterns?
- __main__.py exists?
- templates match actual codebase?

This is why user keeps finding problems that lint doesn't catch. The immune system has a barn-door-sized blind spot in the deployment layer.

Fix (3 agents implementing):
1. init integration tests — pytest catches init regressions
2. verify enhanced — checks deployment completeness (not just file existence)
3. scheduled task self-verify — every metabolic cycle runs verify, issues auto-surface

The lesson: "23/23 lint PASS" gave false confidence. 23 dimensions of knowledge checking + 0 dimensions of deployment checking = the deployment rots undetected.
