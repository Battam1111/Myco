# Trigger-condition list > imperative description for MCP tools

MCP tool descriptions that tell the agent WHEN to call (list of trigger conditions) are dramatically more reliable than descriptions that tell the agent WHAT the tool does. Applied to myco_eat: 'Call when user shares (a) code that worked, (b) decision with reasoning, (c) root cause found after debugging, (d) pasted file/log/URL, (e) TIL/note-to-self phrasing, (f) hard-won knowledge painful to re-derive.' The six-item list reduces the agent's decision surface to pattern-matching rather than free-form judgment.
