# 架构级变更：Myco 是全局的，不是项目级的

## 核心设计变更

Myco 安装在一个固定位置（如 ~/Myco），所有项目共用这一个实例。这就是为什么 upstream 被废弃了——不需要跨项目传递知识，因为知识本来就在同一个地方。

## 当前问题

`myco seed` 当前行为是在目标项目目录里创建所有文件（_canon.yaml、notes/、wiki/ 等），假设 Myco 和项目代码在同一目录。这是错的。

## 需要改的

1. `myco seed` 分两个模式：
   - `myco seed ~/Myco`：安装 Myco 本体（创建完整知识库结构）
   - `myco seed --connect .`：在当前项目中创建 .mcp.json 指向已安装的 Myco

2. 其他项目的 .mcp.json 需要指向 Myco 安装位置的 mcp_server

3. README Quick Start 需要更新为全局安装方式

4. auto-detect 生成的各平台配置也需要指向全局 Myco 路径
