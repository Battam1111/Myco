# 免疫系统盲区：L12 只扫描 .md

## 问题

仿生学重命名后，10 路 Opus 审计仍发现 68+ 处旧名残留。原因：L12 Internal Link Integrity 只扫描 .md 文件，不扫描 .py（字符串常量、docstring）和 .yaml（adapter configs）。

## 未覆盖的问题类别

1. .py 文件中的字符串（annotations.title、docstring、error message）
2. .yaml 文件中的命令引用（adapter configs）
3. 测试文件名与模块名的对应关系
4. CLI dispatch handler 与 subparser 的一致性

## 下一步

1. 扩展 L12 扫描范围：.md + .py + .yaml + .json
2. 新增重命名一致性检查：当模块文件被 git mv 时，自动在所有文件类型中搜索旧名
3. 或者更根本地：实现 `myco rename` 命令，一次性处理模块重命名 + 全文件类型引用更新 + 测试文件重命名
