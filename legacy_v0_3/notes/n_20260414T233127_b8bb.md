# 外部传播定位易错点：把 Myco 错分类到 "memory layer" 赛道

**触发**：Turn 中自承错误——我在讨论 Show HN 定位时把 Myco 描述成 "agent memory substrate + self-evolving"，直接踩进了 MYCO.md 身份锚点明确 disambiguation 的第一个坑（Mem0/Zep 的 memory layer 赛道）。

## 错误模式

把 Myco 说成：
- "agent memory substrate" → 暗示与 Mem0/Letta/Zep 同赛道
- "self-evolving memory" → 只抓住永恒进化律的一小片
- "persistent memory + knowledge graph" → 退化到 A 层库存

## 权威定位（MYCO.md 身份锚点 + substrate_identity_upgrade_craft_2026-04-14.md）

**Agent-First symbiotic cognitive substrate, your agent's other half.**

四词各自承重，缺一不可：
- **Agent-First**：Agent 是唯一操作者，每个表面都为 agent 可发现性设计；人类只用自然语言
- **Symbiotic**：Agent 出智能，Myco 出记忆+代谢+免疫+进化——缺一不可
- **Cognitive**：代谢（七步含淘汰）+验证（29 维 lint）+自我建模（A/B/C/D 四层）+规则进化
- **Substrate**：autopoiesis / selective permeability / temporal organization / co-evolutionary niche 四项结构特征

## 显式否认

Myco **不是**：
- ❌ 记忆层（Mem0/Zep）——那只是 A 层
- ❌ Agent runtime（Hermes/LangGraph）——不管执行流
- ❌ 技能框架（PUA/nuwa）——不规定 agent 怎么做事
- ❌ 基础设施——是 agent 体内器官，不是外部服务

## 对 Show HN / 外传的含义

Myco 开辟的是 "substrate / other-half" 新类目。任何对外文案首次亮相时必须：
1. 先打破 "这是又一个 memory 工具" 的默认分类
2. 用 "agent's other half" / "CPU vs memory-immune-metabolism" 的类比建立新心智模型
3. 显式列出四个 not-is 来对齐预期
