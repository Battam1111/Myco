# Myco Cowork 部署盲点修复

## 发现的 Gap
Myco 的文档和代码（agent_protocol.md、seed_cmd.py、README.md）都假设 `myco connect` 生成的项目级 `.mcp.json` 会被 AI 客户端自动读取。这对 Claude Code、Cursor 等成立，但对 **Cowork 不成立**——Cowork 不读取项目目录的 `.mcp.json`，只通过 Plugin 机制加载 MCP server。

## 修复内容
1. **`docs/agent_protocol.md` §0.5**：新增"三条路径"表格，明确 CLI / 项目 .mcp.json / Cowork Plugin 的持久性和自动注入差异；标注 Cowork Plugin 为唯一可靠的自动注入路径
2. **`src/myco/seed_cmd.py`**：auto-detect 完成后输出 Cowork 警告，提示用户安装 plugin
3. **`README.md`**：Quick Start 下方新增 Cowork 用户专用说明段落

## 不需要修改的
- `open_problems.md`：此 gap 已有完整解决方案（plugin 打包），不属于 open problem
- Claude Code 路径的 `connect` 代码：Claude Code 正确读取项目级 `.mcp.json`，无需警告
- `_COWORK_SKILLS`：skill stubs 仍然有用（为不装 plugin 的用户提供 CLI fallback 指引）
