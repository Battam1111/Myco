# Migration: v0.5.8 → v0.5.9

**Applies to**: any substrate upgrading from Myco v0.5.8 to v0.5.9.

**Summary**: v0.5.9 is a **cleanup-only release** — no contract
changes, no behavioural breaks, no new verbs. It closes the
immune-findings backlog (v0.5.8 self-substrate reported 121 LOW
findings; v0.5.9 reports 0), refines the `DC2` dimension to exempt
`@property` and abstract-protocol overrides, rolls out
`bounded_read_text` DoS protection to the remaining read paths,
ships the canon JSON-Schema for IDE validation, and lands the
v0.5.7 → v0.5.8 migration guide.

---

## TL;DR

```bash
pip install -U myco
```

That's it. No config changes, no script adjustments, no canon edits.

---

## What changed for operators

### Additive

- **Canon JSON-Schema**: `docs/schema/canon.schema.json` ships with
  the package so IDEs (VS Code, JetBrains, Neovim) can validate
  `_canon.yaml` at edit time. See
  [`../schema/README.md`](../schema/README.md) for wiring snippets.
- **Migration guides** (you're reading one): `docs/migration/`
  carries version-to-version upgrade notes starting with
  v0.5.7 → v0.5.8.
- **`bounded_read_text` DoS-cap now covers** canon / notes / dim
  scans / graph reads / propagate source reads / `.gitignore` reads.
  A pathological multi-GB file in any of those paths can no longer
  OOM the kernel.

### Refined (no surprise)

- **DC2 (public-function docstring) exempts `@property` + abstract
  overrides**. Pre-v0.5.9, DC2 over-reported on adapter-protocol
  implementations and dataclass accessors. The rule now matches
  the module docstring's stated intent.
- **Every kernel module now carries a doctrine reference** in its
  module docstring, so `DC4` + `CG1` + `CG2` report clean on
  `myco-self`. Substrates authored against a custom doctrine layout
  may still trip these; that's the intended behaviour.

### Not changed

- R1-R7 text: unchanged.
- Category enum / exit-code ladder: unchanged.
- Manifest verb shapes / mcp_tool names: unchanged.
- `_canon.yaml` shape: unchanged (schema_version stays at `"1"`).
- Dimension roster count (still 25).

---

## Rollback

```bash
pip install 'myco==0.5.8'
```

Your canon works unchanged with either version. The only v0.5.9
additive that lives outside the kernel — the JSON-Schema file under
`docs/schema/` — is inert when absent; nothing breaks.

---

## Why bother upgrading if nothing changed?

Three reasons:

1. **Cleaner lint baseline**. Running `myco immune` on v0.5.9 gives
   you a 0-finding report out of the box (for `myco-self`). Starting
   your own cleanup work from that baseline makes regressions
   visible.
2. **IDE canon validation**. The JSON-Schema catches canon typos
   before `myco hunger` would; faster feedback loop.
3. **Tighter DoS protection**. Every substrate read now has a
   10 MB cap; if you ever ingest from a hostile source, the cap is
   your floor.

No hurry. v0.5.8 is a supported runtime through v0.6+.
