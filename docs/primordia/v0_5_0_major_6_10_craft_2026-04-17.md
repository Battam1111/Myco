# v0.5.0 — 永恒进化 Delivered in Code Craft

> **Date**: 2026-04-17.
> **Layer**: L3 (implementation-level mechanism change) plus L1 rule-4
> text edit and a fresh L3 command_manifest section.
> **Upward**: L0 principle 3 (永恒进化), L1 R1-R7 (unchanged),
> L2 homeostasis (new dimension + `--list`/`--explain` surface),
> L3 command_manifest (4 new verbs) + package_map (new `meta/` package).
> **Governs**: every MAJOR 6-10 closure in the v0.4.1 post-release
> audit (`v0_4_1_audit_craft_2026-04-15.md`).

## Round 1 — 主张 (claim)

v0.5.0 closes the five MAJOR gaps that the v0.4.1 audit left open,
in one pass, so the README's two load-bearing "永恒进化" claims stop
being aspirational:

1. **MAJOR 6**: dimensions register via
   `importlib.metadata.entry_points(group="myco.dimensions")`. The
   hardcoded `ALL` tuple becomes a `_BUILT_IN` fallback for dev
   checkouts. `myco immune --list` and `myco immune --explain <dim>`
   ship.
2. **MAJOR 7**: `test_scaffold.py::PACKAGES` switches to
   `pkgutil.walk_packages`. A new mechanical dimension `MF1` cross-
   checks canon-declared subsystem packages against the filesystem.
3. **MAJOR 8**: `_canon.yaml` with unknown `schema_version` warns,
   does not raise. A `schema_upgraders: dict[str, Callable]`
   registry is added as the forward-compat seam (empty at v0.5).
4. **MAJOR 9**: new verbs `craft`, `bump`, `evolve`. Governance is
   no longer a Markdown-social convention — it is a verb set the
   agent can call.
5. **MAJOR 10**: new verb `scaffold` generates a stub handler for
   any manifest-declared verb. "Adding a verb = edit manifest +
   write Python file by hand" becomes "edit manifest + call
   scaffold".

Single release, sixteen verbs, forward-compatible canon reader,
entry-points-extensible lint kernel. The audit's "doctrine-claimed,
hardcode-blocked" finding becomes doctrine-claimed, hardcode-
unblocked.

## Round 1.5 — 自我反驳 (self-rebuttal)

- **T1**: MAJOR 7 auto-discovering subsystems at runtime is wrong —
  the canon's 5-subsystem block is a deliberate architectural
  commitment, not a gap to be filled by `pkgutil` discovery.
- **T2**: MAJOR 8's `schema_upgraders` dict with no concrete
  v1→v2 upgrader registered is theater — adding a hook that nothing
  uses is UI debt, and shipping `myco canon migrate --dry-run`
  without a real migration to do is worse.
- **T3**: MAJOR 9's `evolve` implies a full mutation/selection
  engine (legacy v0.3 `evolve.py` + `redact.py` + proposal-vs-
  current-state diff harness). Shipping the whole stack in v0.5
  blows the scope.
- **T4**: MAJOR 9's `bump --contract` needs a YAML serializer that
  preserves comments. pyyaml doesn't. ruamel.yaml is a new dep.
  Regex patching is brittle.
- **T5**: MAJOR 10's `scaffold` hits the
  `test_every_handler_resolves` gate — if a verb is added to the
  manifest but its handler is not written at the same commit, the
  test breaks immediately. So `scaffold` can't be what fixes v0.5's
  own new verbs.
- **T6**: MAJOR 6's entry-points discovery returns empty in
  non-`pip install -e .` dev checkouts — local test runs would see
  zero dimensions unless a fallback is wired.
- **T7**: L1 `canon_schema.md` literally says "the immune system
  refuses to read an unknown `schema_version`". Flipping behavior
  without moving doctrine is R7 violation.
- **T8**: `bump` needs to prepend to `contract_changelog.md`. Doing
  this with regex replaces is error-prone if the file has multiple
  `## v` headings with complex prose between them.
- **T9**: Nothing in Myco currently writes to `_canon.yaml` after
  genesis (genesis is one-shot). `bump` is the first mutator — it
  has no precedent to copy.
- **T10**: `Dimension.explain` already returns class `__doc__`.
  Stage B.8 craft said the *module* docstring should become
  `--explain` output. Which one wins?
- **T11**: Is `craft`/`bump`/`evolve`/`scaffold` pre-substrate or
  does it require context?
- **T12**: Tests for these new verbs must validate real artifacts
  (file on disk, canon round-trip, gate verdicts), not just
  "it ran".
- **T13**: Doctrine L2 adds for governance — new file or append?
- **T14**: `meta` subsystem has been a single-file module (`meta.py`).
  `scaffold <verb>` trying to place a handler under `src/myco/meta/`
  would need a special case, which is ugly.
- **T15**: keeping `ALL` as a fallback duplicates state that
  entry-points also declares. Long-term this drifts.
- **T16-T18**: `craft` template shape, `--list` JSON/human duality,
  `bump` version-string validation.

## Round 2 — 修正 (revision)

- **R1** (addresses T1): MAJOR 7 stays but is re-scoped. No runtime
  auto-discovery of subsystems. Instead: (a) switch *the test* from
  a hardcoded list to `pkgutil.walk_packages`, (b) add `MF1`
  dimension that validates canon-declared subsystem packages exist
  on disk. The canon stays declarative (5 subsystems); the filesystem
  is cross-checked, not inferred.
- **R2** (addresses T2): MAJOR 8 ships the `schema_upgraders`
  registry as pure infrastructure — empty at v0.5 — and defers
  `myco canon migrate` to v0.6+ (no UI debt without real use case).
  The flip from raise → warn is the whole MAJOR 8 scope.
- **R3** (addresses T3): MAJOR 9's `evolve` is scope-trimmed to
  **shape validation only** (frontmatter type, title, body size,
  round-marker count, per-round floor). The legacy redact dep and
  mutation engine are explicitly deferred to v0.6+. Five gates, no
  secret-leak gate at v0.5.
- **R4** (addresses T4, T8, T9): `bump` uses regex line-patching on
  the two known top-level fields (`contract_version` and
  `synced_contract_version`). Post-write validation re-reads via
  `load_canon` and restores on parse failure. Changelog insertion
  finds the first `## v` heading and splices above it. No ruamel.
- **R5** (addresses T5): `scaffold` is future-proofing, not a fix
  for v0.5's own new verbs. `craft`, `bump`, `evolve`, `scaffold`
  all ship with real handlers at the same commit as the manifest
  edits.
- **R6** (addresses T6): `discover_dimension_classes()` tries
  entry-points first. On empty result, falls back to the compile-
  time `_BUILT_IN` tuple **and** emits a `DeprecationWarning`
  pointing at `pip install -e .`. Broken entry-points (import
  errors, non-`Dimension` loads) are skipped with a `UserWarning`
  rather than killing the kernel — a single bad plugin must not
  brick lint.
- **R7** (addresses T7): `canon_schema.md` rule 4 is re-authored in
  the same release. The contract_changelog v0.5.0 entry records the
  doctrine change explicitly. No drift.
- **R10** (addresses T10): `Dimension.explain` continues to return
  class `__doc__`. The module-level docstring stays as implementation
  notes. No dual source.
- **R11** (addresses T11): none of the new verbs are pre_substrate.
  `craft` writes into `docs/primordia/`, `bump` mutates canon,
  `evolve` reads from anywhere, `scaffold` writes into `src/`. All
  require a loaded substrate.
- **R13** (addresses T13): no new L2 file. The governance-verbs
  section lives in `L3_IMPLEMENTATION/command_manifest.md`
  alongside the rest of the verb surface. Cleaner.
- **R14** (addresses T14): `meta.py` is promoted to `meta/` package.
  `session_end_run` moves to `meta/session_end.py`. `meta/__init__.py`
  re-exports for backward compat. `scaffold <verb>` now has a
  natural target directory for meta-subsystem verbs — no special
  case. The `from myco.meta import session_end_run` path still
  imports.
- **R15** (addresses T15): `ALL` is explicitly kept as a fallback,
  retitled `_BUILT_IN`, with a comment stating it mirrors
  pyproject's entry-points table. The deprecation warning on
  fallback use makes drift loud.
- **R16-R18**: craft template includes YAML frontmatter + 3-round
  body (Round 1/1.5/2/2.5/3); `--list` JSON/human both ship via the
  existing `--json` global flag; `bump` regex accepts SemVer-ish
  shapes and rejects obvious typos.

## Round 2.5 — 再驳 (counter-rebuttal)

- **T19**: if `meta` becomes a package, `test_scaffold.py`'s
  hardcoded 14-entry PACKAGES list breaks. Fix: also in scope
  (MAJOR 7 switches it to walk_packages).
- **T20**: moving `session_end_run` handler path breaks anyone who
  wrote `handler: "myco.meta:session_end_run"` against the old
  manifest. Myco's own manifest is the only caller; updated in same
  commit. Backward-compat re-export in `meta/__init__.py` covers
  any external pin.
- **T21**: `MF1` will fire on every test that uses a fixture canon
  with `package:` fields pointing under the wrong substrate root.
  Fix: remove `package:` from the genesis template (fresh substrates
  don't claim packages exist); remove from `seeded_substrate`
  fixture. Myco's own `_canon.yaml` keeps `package:` because the
  claim is true there. Tests for `MF1` create their own canons
  targeting existing or missing package paths explicitly.
- **T22**: `bump` is a one-shot verb that should not be chainable
  (calling `bump --contract v0.5.0 --contract v0.5.1` would either
  apply both or be ambiguous). argparse handles this naturally
  (single `--contract` flag, single value). OK.

## Round 3 — 反思 (reflection and decision)

All tensions resolved. Ship order inside v0.5.0:

1. **MAJOR 8 first** — smallest blast radius (one file, one test
   flip). Validates the "make schema forward-compat" idea on its
   own before layering the entry-points work on top.
2. **MAJOR 6** — entry-points discovery + `--list`/`--explain`.
   The fallback path keeps every existing test working during
   development.
3. **MAJOR 7** — `MF1` + `pkgutil` test, after removing `package:`
   from genesis template + fixture (else every existing test with
   `default_registry()` suddenly fires MF1).
4. **Meta package migration** — `meta.py → meta/` plus
   `session_end.py` extracted. Preserves backward-compat re-export.
5. **MAJOR 9** — `craft`, `bump`, `evolve` with tests.
6. **MAJOR 10** — `scaffold` with tests.
7. **Manifest updates** — add four new verbs + `immune --list` /
   `--explain` flags. Update `session-end` handler string.
8. **Doctrine updates** — L1 canon_schema rule 4, L2 homeostasis,
   L3 command_manifest + package_map.
9. **Package metadata** — `pyproject.toml` entry-points +
   `__version__` + `_canon.yaml::contract_version` bump.
10. **Changelogs** — CHANGELOG.md v0.5.0 section,
    contract_changelog.md v0.5.0 section.
11. **Test pass** — full `pytest`, fix any drift.
12. **Release** — git tag v0.5.0, GitHub Release, PyPI upload.

### What this craft revealed

1. The v0.4 audit was correct about exactly which gaps matter. No
   new gaps surfaced during v0.5 design; every MAJOR is closable.
2. The adapter registry (`ingestion/adapters/`) and the dimension
   registry are the two extension points. v0.5 makes the dimension
   side entry-points-native; ingestion adapters already use a
   similar dotted-path import-tolerant pattern and don't strictly
   need an entry-points flip yet (third-party adapter interest is
   lower than third-party dimension interest).
3. The forward-compat schema reader was blocked on one missing
   thing: a warnings channel in `myco.core`. Python's built-in
   `warnings` module covers it. The v0.5 change is small precisely
   because the `Canon.extras` infrastructure was already designed
   for forward-compat — the only lockdown was the raise on
   `schema_version`.
4. `meta.py → meta/` is the real v0.5 architectural move. It makes
   governance a first-class citizen with room to grow. Every future
   agent-callable governance verb lands under `src/myco/meta/` with
   no ambiguity.
