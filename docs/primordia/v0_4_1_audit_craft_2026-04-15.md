# v0.4.1 Post-Release Audit Craft

> **Date**: 2026-04-15
> **Layer**: L4 (substrate-level evidence, not doctrine change)
> **Trigger**: Yanjun asked "does the code actually deliver what the
> README promises?" immediately after the v0.4.1 PyPI release.
> **Method**: four parallel opus sub-agents (250+ tool calls total),
> each auditing one axis: ingestion reach, evolution mechanisms,
> cross-project circulation, and doctrine-vs-README alignment.

---

## Round 1: claim

v0.4.1 shipped a README that says:

> "Myco devours code repositories, framework documentation, datasets,
> papers, chat logs, decisions, and frictions. Anything the agent can
> point at becomes raw material."

> "When the work changes shape, Myco changes with it: the agent
> proposes (craft), you approve, the kernel bumps. New canon fields,
> new lint dimensions, new verbs, new subsystems."

> "You never migrate again."

These are the three load-bearing claims. If they hold, Myco is what
it says it is. If they do not, the README is marketing ahead of
implementation.

## Round 2: refutation (what the audits actually found)

### Ingestion reach (44 tool calls)

`forage` hardcodes 7 text-ish extensions in a frozenset
(`forage.py:21-23`). Every other format, including `.py`, `.pdf`,
`.csv`, `.jsonl`, `.html`, `.ipynb`, `.parquet`, is silently dropped
with no warning. `eat` is a string-writer that takes `--content`
only; it cannot open a file path or fetch a URL. The project has no
HTTP client dependency, no PDF parser, no HTML stripper, no CSV
reader, no code-aware chunker. `symbionts/` contains a 10-line
docstring and zero adapters, zero protocol definition.

Verdict: "devour everything" is aspirational. The code devours UTF-8
strings that the caller has already slurped into memory.

### Evolution mechanisms (97 tool calls)

Adding a canon field works (Canon.extras preserves unknowns). Adding
a lint dimension requires editing 3 sites (file + `__init__.py` ALL
tuple + canon mapping); the promised `--list` and `--explain` CLI
flags do not exist. Adding a subsystem breaks `test_scaffold.py`
(hardcoded 8-package list) and `genesis/templates/canon.yaml.tmpl`
(hardcoded 5 subsystems). `KNOWN_SCHEMA_VERSIONS = frozenset({"1"})`
blocks any future schema. "craft" is a markdown-file convention, not
a verb. No `myco craft`, `myco bump`, or `myco evolve` exists.

Verdict: "self-evolving shape" is aspirational. The architecture
permits evolution in principle but the implementation hardcodes the
current shape in at least 5 places.

### Cross-project circulation (74 tool calls)

`propagate` works end-to-end (verified: source stamping, contract
refusal). `perfuse` is report-only; `--dry-run` is a no-op flag.
The mycelium graph covers `notes/**`, `docs/**`, `_canon.yaml`, and
the entry point; it does NOT cover `src/**` code files. Graph is
in-memory only, never persisted. `symbionts/` has no adapter
protocol. `distill` generates a stub listing source paths and first
lines, then stops; no synthesis, no LLM call. Provenance on intake
is lost (`eat` stamps `source: "agent"` regardless of origin). One
MCP server serves one substrate; no federation.

Verdict: propagate is real. Everything else in the "turn external
content into own advantage" story is a stub or missing.

### Doctrine vs README alignment (35 tool calls)

The five principles are all D-backed (exact quotes in L0_VISION.md).
Three claims are D-extended beyond doctrine:

1. "You never migrate again" (L46 en/zh/ja): doctrine explicitly
   plans for breaking changes (`versioning.md:27`) and the v0.3 to
   v0.4 transition re-exported substrate fresh
   (`canon_schema.md:132-137`).

2. "Even a complete internal rewrite stays a myco release... The
   underlying substrate is never thrown away" (L44): contradicted by
   the v0.3 to v0.4 rewrite's own substrate re-export history.

3. "substrate that ingests frameworks": consistent with L0 principle
   2 ("no filter on what enters") but no doctrine line names
   frameworks as a specific intake target.

No D-contradicted claims found beyond the two historical ones above.
Trilingual parity is clean; zh/ja carry identical claims.

## Round 3: reflection and decision

The architecture is sound. The five principles are real doctrine.
The gap is between doctrine-level permissions and implementation-
level delivery. Specifically:

**BLOCKER gaps (user hits in 10 minutes):**
1. forage silent filter drops code/PDF/CSV/HTML
2. eat needs path and URL support
3. symbionts is empty (no protocol, no adapters)
4. distill is a stub generator
5. no HTTP/PDF/HTML/CSV deps

**MAJOR gaps (doctrine-claimed, hardcode-blocked):**
6. dimension registration is a hardcoded tuple
7. subsystem layout is hardcoded in tests and templates
8. schema version lockdown
9. craft/bump/evolve are not verbs
10. new verb handler must be hand-written

**Plan**: close BLOCKER 1-5 in v0.4.2 (Path B). Close MAJOR 6-10
in v0.5 (Path C). The two D-extended README claims (L44, L46)
remain as directional language with a softer qualifier added once
the forward-compat schema reader (v0.5 scope) is in place.

This craft document is permanent substrate evidence of the audit.
It is not a doctrine change; no contract bump is triggered.
